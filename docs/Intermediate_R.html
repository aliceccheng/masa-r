The dplyr package

The dplyr package includes various functions for common data manipulation tasks, such as filtering rows, selecting specific columns, sorting, modifying or removing columns, and summarising data.

This kind of data manipulation is possible in base R, but dplyr makes data exploration and data analysis easy and fast in R.

If you have not previously installed dplyr package install it using the install.packages()

install.packages("dplyr")

Load the dplyr package using the library() function as shown below:

library(dplyr)

Functions in dplyr

dplyr offers a range of useful functions, each designed to perform a different data manipulation task. These functions are often referred to as “verbs” because they “do things” to data.

Some key dplyr functions are listed in the table below, grouped into categories according to their roles:

Category dplyr function Description
Row operations filter(...) Filters rows (choose rows based on column values).
slice(...) Chooses rows based on location.
distinct(...) Removes duplicate rows.
Row ordering arrange(...) Sorts the data (changes the order of the rows).
desc(...) Indicates descending order when arranging rows - typically used inside arrange().
Column operations select(...) Selects columns to be included in the data.
rename(...) Changes the name of columns.
mutate(...) Creates new columns.
Summarisation group_by(...) Groups the data by the values of one or more columns.
summarise(...) Aggregates data using summary functions (e.g. mean()), reducing it to one row overall or one row per group when used with group_by().
Joins left_join(...)
right_join(...)
inner_join(...)
full_join(...)
Other functions across(...) Applies one or more functions to selected columns (often inside summarise() or mutate()), letting you transform or summarise many columns in one go.
coalesce(...) Returns the first non-missing (NA-free) value from a set of vectors or columns, element by element, effectively “filling in” missing values from left to right.
if_else(...) Evaluates a logical test and returns one value where the test is true and another where it is false.
case_when(...) Evaluates multiple conditions in order and returns a value for the first condition that is true, allowing you to recode or create variables with many conditional rules.

General syntax and the pipe operator: %>%

All of the dplyr functions take a data frame (or tibble) as the first argument. 

For example, in the select() command below, the first argument is penguins, telling R that we want to apply the select() function to our penguins data frame. (Specifically, this command selects the species and island columns.)

select(penguins, species, island)

However, we could use the following code to achieve the same thing:

penguins %>% select(species, island)

In this version, the name of the data frame (penguins) is taken outside the function, and followed by ‘%>%’ before the function call.

%>% is called the pipe operator and comes from the magrittr package, which is included in dplyr. (Recall that an operator in R is a symbol that performs an operation (action) on values or objects.)

The pipe operator passes the result of one function as the first argument to the next function; the result from one step is then “piped” into the next step.

An easier way to think about it is to read ‘%>%’ as ‘then’. So in the example above, the code is telling R to: take the penguins data frame, then, select the species and island columns.

The real benefit of using %>% comes when combining multiple dplyr functions together. It allows for cleaner and more readable code by avoiding nested function calls or the need to save intermediate objects - more on this later.

We will start by using single dplyr functions, but will still use the %>% version to keep the syntax consistent.

Row operations

The filter(...) function allows you to select a subset of rows in a data frame.

NoteWhen is filtering useful?

Filtering is useful when…

The first argument is the name of the data frame you want to work with.

The second and subsequent arguments refer to variables within that data frame, selecting rows where the expression is true. Write better

General syntax:

filter(data, condition1, condition2, ...)

For example, filter for penguins with body mass greater than 4000g. R checks every row of the data frame penguins to see whether the condition body_mass_g > 4000 to…. and return all rows where TRUE (explain).

filter(penguins, body_mass_g > 4000)

Further examples:

If you want to check for equality…..

filter(penguins, Species == "Adelie")
filter(penguins, year == "2007")

Multiple conditions………

filter(penguins, Species == "Adelie", body_mass_g > 4000)
